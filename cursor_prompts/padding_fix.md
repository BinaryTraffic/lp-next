## Cursor修正依頼：余白（padding）の反映

### 問題
元画像（219×51px）には上下にグレーの余白がある。
現在の書き出しはボタンが全面に描画されており余白がない。

### 期待する出力
```
元画像:
┌─────────────────────┐ ← グレー余白
│ ┌─────────────────┐ │
│ │  LINE 友だち追加 │ │ ← 実ボタン
│ └─────────────────┘ │
└─────────────────────┘ ← グレー余白

書き出し（現状）:
┌─────────────────────┐
│  LINE 友だち追加     │ ← 余白なし・全面ボタン
└─────────────────────┘

書き出し（期待）:
┌─────────────────────┐ ← グレー余白（元と同じpx）
│ ┌─────────────────┐ │
│ │  LINE 友だち追加 │ │
│ └─────────────────┘ │
└─────────────────────┘ ← グレー余白（元と同じpx）
```

### 修正方針
image_composite.php に以下を実装する：

1. 元画像（source_url）をGDで読み込む
2. 上辺から走査：R,G,B が全て200以上の行 = 余白行
3. 下辺から走査：同条件
4. 左辺・右辺も同様に走査
5. padding_top / padding_bottom / padding_left / padding_right（px）を算出
6. 最終キャンバスサイズ = 元画像と同じ width × height
7. 余白部分は元画像の余白色（サンプリング）で塗りつぶし
8. 実ボタン領域（余白を除いた内側）に合成画像を配置

### 入力JSONに source_url を追加
```json
{
  "source_url": "/output/assets/img/btn01.jpg",  ← 元画像（余白検出用）
  "background_url": "/output/ai_images/hf_xxx.jpg",
  "width": 219,
  "height": 51,
  "texts": [...],
  "icons": [...]
}
```

### 制約
- PHP 8.x + GD
- Node.js・Composer 使用不可
- テストはApache経由で行うこと（php -Sは権限エラーになる）
