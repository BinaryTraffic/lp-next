# 開発専用（`dev/`）

**本番デプロイの対象外**を原則とする置き場です。補助スクリプト・検証用 HTML・モック画像・Figma エクスポート下書きなど、**顧客向けURLに乗せたくない**／**リポジトリに載せたくない**作業用を分離します。

| 下位 | 用途 |
|------|------|
| **`scripts/`** | ワンショット変換、データ検査、ローカルだけで動かす小スクリプト（シェル／Node 等。用途ごとにサブフォルダ可） |
| **`images/`** | デザイン仮置き、プレースホルダ、スクリーンショット下書き（**本番LPやルート用には使わない**想定） |
| **`images/_local/`** | 大容量・社外秘のモック。Git 管理外（[`.gitignore` ](../../.gitignore)）。必要ならここにだけ置く |

## 本番用との棲み分け

- **本番・ステージングに配信する**静的ファイル（`index.html` や `journal.html` から参照する画像・JS）は **[`../assets/`](../assets/)** を使う。`lp_reverse_cms` 管理画面用の既存CSS/JSは従来どおり **`lp_reverse_cms/assets/`**。
- この `dev/` を Git に含めていても、**デプロイ作業**ではディレクトリごと同期しない、または同パスに置かない運用にすると混線が減ります（例はジャーナル「開発/デプロイの棲み分け」節を参照）。

## セキュリティ

**DocumentRoot が `current/` 全体**の場合、このフォルダも URL 経由で見える可能性があります。同梱の **`.htaccess` は Apache 向け**（当ディレクトリへの直接アクセスを拒否）。Nginx では別途 `location` で拒否するか、**デプロイ先に `dev/` をコピーしない**運用にしてください。PHP ビルトインサーバーでは `.htaccess` は効きません（ローカル想定のため、公開サーバーと同列にしないこと）。
